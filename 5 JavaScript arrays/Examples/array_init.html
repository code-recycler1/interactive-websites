Array Creation Methods
Literal Syntax (Preferred)
let empty = []; // Empty array
let tens = [10, 20, 30, 40, 50]; // Numeric array
let mixed = ['Hello', 'World', 10, 20, true]; // Mixed types
Constructor (Avoid)
let emptyArray = new Array(); // Avoid - use [] instead
let sparse = [1, , 3]; // ⚠️ Sparse array - AVOID
let bad = new Array(5); // Creates holes!
Avoid sparse arrays! They cause inconsistent iteration behavior.
Accessing and Modifying Elements
Zero-Based Indexing
let a = tens[2]; // a = 30 (third element)
Adding Elements
tens[5] = 60; // Appends 60 at the end
Modifying Elements
tens[0] = 70; // Changes first element to 70
Warning: Adding elements with non-consecutive indices creates sparse arrays!

⚠️ SPARSE ARRAYS - AVOID
Sparse arrays have missing indices (holes). They cause:
  • Unexpected behavior with iteration methods (forEach skips holes)
  • Performance degradation
  • Confusing debugging
BAD - Creates holes
let bad = [1, , 3]; // Sparse - hole at index 1
let bad2 = new Array(5); // Sparse - all indices are holes
GOOD - Dense arrays
let good = [1, 2, 3]; // Dense - all indices have values
let good2 = Array(5).fill(0); // Dense - all filled with 0
✅ Recommended Patterns
Empty to Fill
const arr = [];
arr.push('first');
Pre-sized with fill()
new Array(5).fill(0)
Generated Values
Array.from({length:5}, (_,i)=>i*10)
Shallow Copy
const copy = [...original];

📚 FINAL KEY TAKEAWAYS:
  • ✅ ALWAYS use array literal syntax [] over new Array()
  • ✅ Arrays are zero-indexed: first element is at index 0
  • ✅ Length property returns number of elements (highest index + 1)
  • ✅ Setting length to a smaller value truncates the array
  • ✅ Setting length to 0 empties the array
  • ⚠️ Setting length to a larger value creates empty slots (sparse array - AVOID)
  • ⚠️ Avoid sparse arrays (holes) - they cause inconsistent iteration behavior
  • ✅ Use Array.fill() or Array.from() for pre-sized dense arrays
  • ✅ JavaScript arrays are dynamically sized and can hold mixed types
  • ✅ Always validate indices before access to avoid undefined values
