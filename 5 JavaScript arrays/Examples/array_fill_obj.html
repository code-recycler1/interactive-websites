Array.fill()

Understanding reference vs value behavior when filling arrays

Key Takeaways

Primitives (Safe)
fill() with Numbers, Strings, Booleans

Primitives are copied by value - each element gets its own independent copy.

Example: new Array(10).fill(0) creates 10 independent zeros
Objects - CRITICAL PITFALL
fill() with Objects Creates Shared References

All elements reference the same object in memory. Modifying one affects all!

Common Bug: new Array(3).fill({type: 'student'}) creates shared references
Solution Pattern 1
fill(null).map() → Unique Objects

Fill with null (primitive) first, then map to create unique object instances.

new Array(3).fill(null).map(() => ({
    type: 'student'
}));
Solution Pattern 2 (Preferred)
Array.from() - Cleaner Pattern

Single-pass creation without intermediate array. More readable and concise.

Array.from({ length: 10 }, () => ({
    type: 'student'
}));
Additional Context
forEach() - Iterating Arrays

Complementary method for iterating through array elements after creation.

let fruits = ['Apple', 'Banana'];
fruits.forEach((item, index) => {
    console.log(`Index ${index}: ${item}`);
});
