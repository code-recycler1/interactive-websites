Quick Reference
Method Syntax Description Example
join() array.join(separator) Converts array to string [1,2,3].join('-') → "1-2-3"
split() string.split(separator, limit) Converts string to array "a,b,c".split(',') → ["a","b","c"]
💡 Key Takeaways:
  • join() is the inverse of split() (with matching separators)
  • Default separator for join() is comma (,)
  • split() with empty string creates character array
  • join() is optimized for performance - use instead of manual concatenation
  • Numbers become strings when using split() - be aware of type conversion
  • null and undefined become empty strings in join()
⚠️ Common Pitfalls:
  • Split always returns strings - numbers, booleans become strings
  • Empty array join returns empty string, not null
  • Split with empty string on empty string returns [""], not []
  • Separators within data need escaping or different delimiter
  • Regex separators can have unexpected behavior with special characters
Example Outputs (Console)
join() Examples
myArray.join() // "10,20,30,40,50"
myArray.join("*") // "10*20*30*40*50"
myArray.join(" ") // "10 20 30 40 50"
myArray.join("") // "1020304050"
split() Examples
"a,b,c".split(",") // ["a","b","c"]
"a b c".split(" ") // ["a","b","c"]
"apple,banana,orange".split(",", 2) // ["apple","banana"]
