Array.filter() Method

Filter ages to show only adults (18 years or older)

Interactive Demo
Click the button to see filtered results
Data Source
Original Ages Array
const ages = [32, 33, 16, 40, 7, 56];

Using const for immutability reference (ES6+ best practice)

Console output: Original ages array: [32, 33, 16, 40, 7, 56]
Filter Callback Function
Predicate Function: isAdult()
function isAdult(age) {
    return age >= 18;
}

Returns true for elements to include, false to exclude

Alternative: Arrow function syntax
const isAdult = (age) => age >= 18;
How filter() Works
Key Concepts
  • Creates a NEW array (does NOT modify original)
  • Callback invoked for each element in order
  • Returns shallow copy of elements that pass the test
  • Empty slots in sparse arrays are skipped
Alternative Patterns
Different Filter Callback Styles
Inline arrow function:
const adults = ages.filter(age => age >= 18);
External predicate:
const adults = ages.filter(isAdult);
Multiple criteria:
const adultsUnder50 = ages.filter(age => age >= 18 && age < 50);
Additional Filter Examples
Filter Even Numbers
numbers.filter(n => n % 2 === 0)

[1,2,3,4,5,6] → [2,4,6]

Filter Active Users
users.filter(user => user.active)

Returns only users with active: true

Remove Falsy Values
mixed.filter(Boolean)

Removes false, null, undefined, 0, ""

Quick Reference
Method Description Returns Modifies Original?
filter() Creates new array with elements that pass the test New Array No
map() Creates new array with transformed elements New Array No
forEach() Executes function for each element undefined No (but can side-effect)
reduce() Reduces array to single value Single Value No
Key Takeaways:
  • filter() returns a NEW array — original array is immutable
  • Callback must return boolean: true = include, false = exclude
  • Use const for arrays that shouldn't be reassigned
  • Prefer addEventListener over inline onclick for maintainability
  • Empty slots in sparse arrays are ignored (callback not invoked)
