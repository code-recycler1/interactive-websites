Key Concepts
What is forEach()?

forEach() executes a provided function once for each array element.

Callback Parameters:
  • currentValue - The current element being processed
  • index (optional) - The index of the current element
  • array (optional) - The array being traversed 
array.forEach((value, index, array) => {
    // your code here
});

Common Pitfalls
Cannot break or return early
forEach() always iterates through the entire array. Use for...of if you need early termination.
DOM updates inside loops
Updating DOM inside forEach causes layout thrashing. Calculate first, then update once.
Async operations don't wait
forEach() doesn't handle async/await. Use Promise.all() with map() instead.
Best Practices
  • Use forEach() for side effects (logging, DOM updates after calculation)
  • Separate calculation from presentation - calculate sum first, then update DOM once
  • Use map()/filter()/reduce() for transformations instead of forEach with accumulators
  • Use for...of when you need break/continue or async/await
  • Keep callback functions pure when possible (no external side effects)
forEach() vs Other Iteration Methods
Method Returns Modifies Original? Early Exit Best Use Case
forEach() undefined No ✗ No Side effects, aggregation
map() New Array No ✗ No Transform each element
filter() New Array No ✗ No Create subset based on condition
reduce() Single Value No ✗ No Accumulate to single value
for...of undefined No ✓ Yes Early exit, async/await
find() First match No ✓ Yes Get first matching element
Alternative Patterns & Examples
Sum with reduce()
numbers.reduce((acc, curr) => acc + curr, 0)

More semantic than forEach for summing

Transform with map()
numbers.map(num => num * 2)

Returns new transformed array

Early exit with for...of
for (const num of numbers) { if (num > 20) break; }

Supports break/continue

📚 Key Takeaways:
  • forEach() executes a function once for each array element
  • Cannot break or return early - use for...of if needed
  • Callback receives up to 3 arguments: value, index, array
  • Always separate calculation from DOM updates for better performance
  • For summing numbers, reduce() is more semantic than forEach()
  • Use const for arrays that won't be reassigned
